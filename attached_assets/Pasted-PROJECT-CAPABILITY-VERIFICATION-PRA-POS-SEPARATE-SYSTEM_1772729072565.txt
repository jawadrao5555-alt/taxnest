Context:
I already have a production Laravel system called NestPay which handles FBR Digital Invoicing.

IMPORTANT REQUIREMENT:
The new project must be completely independent and must NOT modify, connect to, share database with, or access the existing NestPay system in any way.

Goal:
Build a new Laravel application called PRA POS for Punjab Revenue Authority POS reporting.

Before development starts, confirm that the Replit environment can support the following architecture.

Verification Points:

1. Separate Laravel Application
Confirm that a completely separate Laravel project can run in the same Replit workspace or environment without touching the existing NestPay project.

Example structure:
/
├── nestpay/        (existing production project – DO NOT MODIFY)
└── pra-pos/        (new Laravel PRA POS project)

2. Independent Database
Confirm that the new PRA POS project can use its own database connection and tables independent from NestPay.

Example:
nestpay_db
pra_pos_db

- controllers
- services
- Blade views
- API integrations

Example:
Http::withToken(...)->post(...)

pra_logs

6. POS Interface
Confirm that Replit can run a web POS interface including:
- billing screen
- item/service list
- tax calculation
- receipt preview
- print support

7. Multi-Company Support
Confirm that the system can support multi-tenant architecture where each company stores:

pra_api_token
pra_api_url
terminal_id

8. Project Isolation Guarantee
Confirm that the new PRA POS project can remain completely isolated so that:
- no NestPay files are modified
- no shared database tables are used
